OnCall for Nagios

About


Version:  2.0.2 


OnCall for Nagios works with iPhone 4S, iPhone 4, iPhone 3GS, iPhone 3G, the original iPhone, and iPod touch.

OnCall for Nagios is a simple, intuitive, and powerful interface to your existing Nagios installation.

Setup is easy - just enter your Nagios url (and optional username/password for HTTP authentication) - it just works. There's no need for additional server side configuration.

Features

  • summary view of problems with highlights for warning/critical/acknowledged issues
  • multiple nagios instance settings
  • ability to see service level detail
  • New perform actions on a host with quick access from problems view
  • easy to see status coloring and notification state from problems view
  • New ability to filter the services shown on the problems view based on state
  • ability to quickly acknowledge an issue with a user specified default message
  • New ability to disable notifications in the detailed acknowledgements dialogue
  • swipe to acknowledge on problems view
  • ability to acknowledge an issue with a custom, one-time, message
  • ability to reschedule another check of a service
  • ability to schedule downtime for a service
  • Updated browse hosts by all, service-group, and host-group

Frequently Answered Questions

#1

It's not working and I know my settings are right...
 

OnCall for Nagios is pretty good at letting you know when an HTTP request it's making failed.

If there isn't any obvious error message presented, check the following items:

  1. Make sure the username and password you entered is correct.
  2. Make sure that the URL you've entered is to the /cgi-bin/ directory for the given Nagios installation.
  3. Check the Nagios web-server logs to ensure it is seeing the requests from the app and they are returning the 200 HTTP response code.
